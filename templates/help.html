このアプリは

  • スマホからシフト希望を出す（職員）
  • 全員の希望を確認して調整する（管理者）
  • シフト入力ができるのは 日・月・木 のみ（開室曜日）

ログイン

  1. 管理者から渡された ID と 一時パスワード でログイン
  2. 初回は パスワード変更画面に自動で移動。新しいパスワードを設定するまで他画面に進めません
  3. 2 回目以降にパスワードを変えたいときは、メニューの 「パスワード変更」 から
何度も間違えると 1 分間ログインできなくなります（連続失敗のレート制限）。少し待ってからもう一度お試しください。

シフト希望の出し方

  1. メニュー → 「自分のシフト入力」
  2. 今月か翌月を選ぶ
  3. 日・月・木の各日付について、出勤できる日は 〇、できない日は ×（既定はすべて 〇）
  4. 必要なら 「備考」 で一言メモを残す（例：午前のみ可）
  5. 「この内容で保存する」 で送信
  6. 送信後、続けて翌月分を出すか聞かれます

備考は管理者だけが見られます。シフトに条件があるとき（時間帯、体調など）にお使いください。

管理者の操作

全体のシフト確認

その月に「〇」を出した人をカレンダーで一覧。名前タグをタップで備考が見られます。

シフト入力（管理者）

管理者自身のシフト希望を入力します（職員と同じ画面）。

ユーザー管理

  • 追加: ID・一時パスワード・お名前・色・権限を入れて「登録」。本人は初回ログイン時にパスワード変更を求められます
  • 修正: 一覧の「修正」で値が入ります。ID は変更不可。パスワード空欄なら据え置き、入力すると本人は次回再変更を求められます
  • 停止 / 復活: 退職・休職は「停止」、戻ったら「復活」。停止中はログイン不可、シフト履歴は残ります
削除ボタンは置いていません。 退職は「停止」で十分です（履歴が残るため後で確認できます）。 どうしても物理削除が必要な場合のみ、バックアップを取った上で管理者が sqlite3 で行う運用です（README 参照）。
安全装置:
  • 自分自身を降格・停止できません
  • 最後の有効な管理者は降格・停止できません
  • 修正時に ID は書き換えられません（別人を新規作成してしまう事故を防ぐため）
  • 同じ表示名のユーザーは作れません

アプリを初めて立ち上げる方へ（管理者向け）

試用開始までの流れは README の「試用開始までの流れ（半日）」セクションに 5 ステップでまとめています：

  1. ローカルで動作確認（5 分）
  2. 本番サーバー準備：環境変数・DB フォルダ・gunicorn・HTTPS（30 分）
  3. 実機での目視チェック + バックアップ復元演習（30 分）
  4. 関係者へ /help（このページ）を共有（5 分）
  5. 3〜5 名で 1 か月の試用開始
運用上の注意:
  • 初期 admin パスワードは控えたら、起動ログから消去してください
  • cron で日次バックアップを設定し、バックアップファイルも chmod 600 で保護
  • 毎週ログとバックアップを確認、フィードバックを記録

困ったとき

  • ログインできない → ID とパスワードを管理者に確認。停止中の可能性も
  • 「ID または パスワードが正しくありません」が続く → 1 分待ってから再試行
  • ボタンが押せない → ログインし直してください（セッション切れ）
  • お名前に保存できない文字 → / \ ? # & < > と改行は使えません
