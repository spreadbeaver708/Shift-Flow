このアプリは

  • 職員はスマホからシフト希望を出す
  • 管理者は全員の希望を確認して調整する
  • 入力できるのは 日・月・木（開室日）だけ

ログイン

  1. 管理者から渡された ID と 一時パスワード でログイン
  2. 初回は パスワード変更画面に自動で移動。新しいパスワード（8 文字以上）を決めるまで先に進めません
  3. あとで変えたいときは、メニューの 「パスワード変更」 から
何度も間違えると 1 分間ログインできなくなります。少し待ってからもう一度お試しください。

シフト希望の出し方

  1. メニュー → 「自分のシフト入力」
  2. 今月か翌月を選ぶ
  3. 日・月・木の各日について、出られる日は 〇、出られない日は ×（はじめは全部 〇）
  4. 必要なら 「備考」 に一言メモ（例：午前のみ可）
  5. 「この内容で保存する」 で送信。続けて翌月分も出せます
備考は 管理者だけ が見ます。相談内容や個人情報は書かないでください。勤務の条件（時間帯・体調など）だけにしましょう。

確定シフトの確認

メニューの 「確定シフトを見る」 で、管理者が決めた確定シフト（出勤日）をカレンダーで確認できます。希望ではなく、確定した出勤予定です（読み取り専用）。

管理者の操作

全体のシフト希望を確認

その月に「〇」を出した人をカレンダーで一覧。名前タグをタップすると備考が見られます。

提出状況の一覧

その月に誰が希望を提出済み／未提出かを一覧で確認できます。未提出の人がいればひと目で分かります。

確定シフトの作成/編集

職員を選んで、その人の確定シフト（出勤日）を 〇／× で編集します（初期値は ×＝休み）。各日には本人の「希望」も下敷きで表示されるので、希望を見ながら確定できます。確定した内容は職員全員が「確定シフトを見る」で確認できます。

操作ログ

ログインやシフト提出、ユーザー操作などの記録を新しい順で確認できます。パスワードや備考の本文は記録していません。

シフト入力（管理者）

管理者自身の希望を、職員と同じ画面で入力できます。

ユーザー管理

  • 追加: ID・一時パスワード・名前・色・権限を入れて「登録」。本人は初回ログインでパスワードを変更します
  • 修正: 一覧の「修正」で値が入ります。ID は変更不可。パスワードは空欄なら据え置き
  • 停止 / 復活: 退職・休職は「停止」、戻ったら「復活」。停止中はログイン不可で、履歴は残ります
削除ボタンはありません。 退職は「停止」で十分です（履歴が残ります）。
安全装置: 自分自身や最後の管理者は降格・停止できません。ID の書き換えや表示名の重複もできません。

運用のポイント（管理者）

  • 週に 1 回、自分でログインして動作を確認
  • Render のディスクは 毎日自動でスナップショット（7 日保管）されますが、データベースの復元にはスナップショットを使わないでください（破損復元の恐れ）。DB のバックアップ・復元は Render の「Shell」から sqlite3 ... ".backup" で取得したファイルを使い、取得後は PRAGMA integrity_check; で確認します
  • 試用中の 気づき・要望をメモしておく（続けるかの判断材料に）

くわしい運用は README を参照してください。

困ったとき

  • ログインできない → ID とパスワードを管理者に確認（停止中かも）
  • エラーが続く → 1 分待ってから再試行
  • ボタンが押せない／画面が戻る → ログインし直してください（セッション切れ）
  • 名前に使えない文字 → / \ ? # & < > と改行は使えません
